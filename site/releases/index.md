# Platform Journey Overview

Azawslab is presented as a staged enterprise platform journey. Each release builds on the previous one and adds a higher level of engineering maturity, operational control, and evidence.

## Release path

| Stage | Capability | Start here |
|---|---|---|
| Release 1 | Hybrid Modern Workplace, Identity, Endpoint Security and Microsoft 365 Operations | [Release 1 summary](release1.md) |
| Release 2 | Azure Platform Engineering, Security, Automation, Private Platform and AI Operations | [Release 2 summary](release2.md) |
| Release 3 | Multi-Cloud Kubernetes, GitOps and DevSecOps Roadmap | [Release 3 roadmap](release3.md) |

## What the journey demonstrates

| Area | Reviewer signal |
|---|---|
| Identity and workplace operations | Hybrid identity, Conditional Access, Intune, Autopilot, Microsoft 365, Purview, Graph and PowerShell operations |
| Azure platform engineering | Terraform state separation, OIDC delivery, governance, networking, private platform delivery and evidence-backed operations |
| Automation and operations | Ansible, AWX, validation, monitoring, backup, recovery and controlled Day 2 execution |
| Hybrid and multi-cloud networking | BGP, VPN, firewall/NVA routing, Azure/AWS branch patterns and secure transit |
| Private platform and secure workspace | Private AKS, AVD, FSLogix and private access design |
| AI-assisted CloudOps | Governed AI operations enclave with policy mediation and human-reviewed execution |
