# Diagrams

This folder contains the main visual artifacts used to explain the architecture, control flow, roadmap, and evidence structure of the `azawslab Enterprise Hybrid Security Platform`.

## What this folder proves

- Release 1 has a documented end-state architecture, not just screenshots
- identity, messaging, endpoint control, and recovery were mapped visually
- the project is structured as a phased platform across Releases 1, 2, and 3
- key control and implementation relationships were explained in diagram form

## Key diagrams

- `01-release1-end-state-architecture.png` - high-level Release 1 platform view
- `02-identity-messaging-endpoint-control-flow.png` - identity, Exchange, Intune, and access-control relationship map
- `03-bitlocker-recovery-and-stale-device-cleanup-scenario.png` - recovery and cleanup scenario
- `04-phased-roadmap-release1-release2-release3.png` - three-release roadmap
- `05-release1-implementation-flow-and-proof-map.png` - Release 1 proof and implementation flow
- `06-release1-security-and-compliance-mapping.png` - Release 1 control-domain map
