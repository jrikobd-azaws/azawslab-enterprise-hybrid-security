# Foundation Documentation

This folder contains cross‑release architecture, business context, and capability mapping for the `azawslab Enterprise Hybrid Security Platform`.

It explains **why** the platform exists, **what** it looks like across three releases, and **how** the implementation is structured as a staged, evidence‑backed engineering journey.

---

## Files in This Folder

| File | Purpose |
|------|---------|
| `00-business-scenario.md` | Fictional enterprise scenario – the problem statement behind the platform |
| `01-platform-overview.md` | High‑level platform shape, three‑release structure, and architectural layers |
| `02-current-state-architecture.md` | Starting on‑premises environment before Release 1 implementation |
| `03-target-state-architecture.md` | Intended end state after Release 1 (baseline + advanced validation) |
| `04-roadmap.md` | Staged progression from Release 1 to Release 2 (Azure) to Release 3 (workloads) |
| `05-skills-and-evidence-index.md` | Capability‑to‑proof ledger for recruiters, hiring managers, and reviewers |
