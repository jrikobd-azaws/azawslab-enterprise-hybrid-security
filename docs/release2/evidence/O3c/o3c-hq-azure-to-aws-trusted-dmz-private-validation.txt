O3c private data-plane validation - Azure/HQ to AWS trusted and DMZ
===================================================================

Purpose
-------

Validate O3c segmented private routing across:

- HQ / VyOS
- Azure VPN Gateway BGP transit
- AWS Cisco 8000V
- AWS trusted subnet
- AWS DMZ subnet negative control

Expected result:

- AWS trusted private VM is reachable.
- AWS DMZ private VM is not reachable.
- This validates segmented route propagation and security group behavior.
- This does not claim FortiGate inspection for AWS flows.

---------------

Conclusion
----------

O3c private route segmentation is validated:

- Trusted AWS prefix is reachable over private hybrid/multi-cloud routing.
- DMZ AWS subnet remains unreachable as a negative control.
- Cisco advertises only the approved trusted AWS branch prefix.
- Full VPC and DMZ advertisement are not used for this proof.

Non-claim
---------

This evidence does not claim FortiGate inspection for AWS flows. FortiGate inspection can only be claimed after FortiGate route path, policy counters, or logs prove traversal.
